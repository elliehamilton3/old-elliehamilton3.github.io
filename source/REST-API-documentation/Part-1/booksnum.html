books/<book-id>

URI Template

http://waisvm-rfp1.ecs.soton.ac.uk/books/<book-id>

URI Parameters

book-id

required

The URI Parameter <book-id> takes a number which corresponds to the id assigned to the book. This is automatically assigned by the system to be the first available id in the system which is not already being used. This allows for id's which may have been freed up following the delete of a book.

Example value: 130

HTTP Methods

GET, PUT, PATCH, DELETE, HEAD, OPTIONS

Response Formats

Renders Formats XML, JSON, TEXT/CSV
Parses Formats XML, JSON, TEXT/CSV
Requires Authentication? Yes - for PUT, PATCH and DELETE requests

DELETE /books/130

Deletes the book located at /books/130 if the user meets the required permissions i.e. if they are the owner of the book/they created the book.

HTTP Status Code: 204 NO CONTENT or if the correct authorization token id not given then returns 403 FORBIDDEN

users/<user-id>

URI Template

http://waisvm-rfp1.ecs.soton.ac.uk/users/<user-id>

URI Parameters

user-id

required

The URI Parameter <user-id> takes a number which corresponds to the id assigned to the user. This is automatically assigned by the system to be the first available id in the system which is not already being used. This allows for id's which may have been freed up following the delete of a user.

Example value: 2

HTTP Methods

GET, PUT, PATCH, DELETE, HEAD, OPTIONS

Response Formats

Renders Formats XML, JSON, TEXT/CSV
Parses Formats XML, JSON, TEXT/CSV
Requires Authentication? Yes - for PUT, PATCH and DELETE requests

Example Output

GET /users/1

Returns information about the user located at /users/1 with all of the information fields including a URL link to the users loans page which is automatically created similar to the way the users/<user-id> page is on creation of a new user.

HTTP Status Code: 200 OK

Headers

Accept: application/xml; (Default)

    <loans>http://waisvm-rfp1.ecs.soton.ac.uk/users/1/loans</loans>
    <name>Ellie H</name>
    <owner>eh14g13</owner>

DELETE /users/130

Deletes the user located at /users/130 if the user meets the required permissions i.e. if they are the owner of the user/they created the user. Also deletes all of following/associated pages at /loans and /loans/<loan-id>.

HTTP Status Code: 204 NO CONTENT or if the correct authorization token id not given then returns 403 FORBIDDEN

Example Input (for modifying a resource)

PUT /users/130

Use PUT when you can update a resource completely through a specific resource. For instance, if you know that a user resides at http://waisvm-rfp1.ecs.soton.ac.uk/users/130, you can PUT a new resource representation of this article directly through a PUT on this URL.

For users/<user-id> PUT and PATCH requests work the same because a user only has one field which is set and that is 'name'.
